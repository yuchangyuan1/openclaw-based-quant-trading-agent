# OpenClaw-based Quant Trading Advisor

一个基于 **OpenClaw** 的低频量化交易咨询 Agent 项目，面向中低频（以日频 / 周频为主）的股票投资决策支持。

> 目标：提供可解释、可复盘、重风险控制的市场观察与投资建议，而不是自动下单交易系统。

---

## 项目定位

该项目实现了一个“量化分析 + 投资建议 + 飞书推送”的工作流：

- 定时抓取市场与个股数据
- 计算低频量化信号与风险评分
- 输出结构化中文建议
- 推送日报 / 周报 / 异动提醒到飞书

适用人群：关注 A 股、偏中性稳健风险偏好的中低频投资者。

---

## 核心原则

来自项目人格与策略约束（见 `SOUL.md`）：

- 可解释优先于复杂
- 风险控制优先于收益冲动
- 没有数据就不下结论
- 没有显著变化就不重复打扰
- 给建议时必须说明为什么

---

## Agent 分工

见 `AGENTS.md`：

- **MarketScanner**
  - 抓取大盘、行业、股票池、财报、新闻、公告
  - 输出标准化市场数据包

- **SignalAnalyst**
  - 计算低频量化指标
  - 输出候选信号与风险评分（`signal_report.json`）

- **Advisor**
  - 将量化结果转换为中文投资建议
  - 每条结论附：理由、风险、置信度

- **Notifier**
  - 整理为飞书可读格式
  - 调用飞书机器人推送

---

## 输出格式（默认）

见 `USER.md`：

1. 市场概览  
2. 重点信号  
3. 股票池建议  
4. 风险提示  
5. 操作建议（观察 / 持有 / 减仓 / 增持）

---

## 调度计划（Heartbeat）

见 `HEARTBEAT.md`：

- **每个交易日 08:30**：盘前观察（隔夜全球市场 + 股票池新闻公告）
- **每个交易日 15:30**：收盘分析（日报推送）
- **每周六 10:00**：周报（收益 / 回撤 / 信号变化汇总）

---

## 工具抽象（逻辑层）

见 `TOOLS.md`：

- `market_data`：行情与基本面抓取
- `quant_analysis`：因子计算、信号生成、风险评估
- `feishu_bot`：飞书推送（先摘要后全文）

约束：

- 数据必须标注时间
- 无法获取新数据时不得编造
- 建议必须有依据

## 边界声明

本项目是**投资咨询辅助系统**，不是交易执行系统：

- 不直接执行交易
- 不承诺收益
- 不提供高频交易决策
- 不输出无依据结论
