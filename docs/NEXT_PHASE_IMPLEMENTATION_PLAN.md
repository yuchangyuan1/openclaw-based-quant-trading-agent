# 后续项目实施计划表（V2）

## 目标
在保留 Tushare 主链路前提下，引入 Akshare 回退与置信度联动，提升数据可用性与建议可信度。

## 阶段与任务

### P0（今天）
1. 字段级数据源标注（tushare/akshare/none）
2. 置信度联动：
   - 财务缺失降级
   - 使用回退源小幅降级
   - 数据质量闸门降级
3. 日报显示“财务因子可用率 + 回退比例”

### P1（1-2天）
4. `fundamental_probe` 探针脚本（逐接口可用性报告）
5. 周报增加“数据源稳定性”板块
6. 异常告警：连续N天 fundamental=error 触发系统状态提醒

### P2（3-5天）
7. 因子权重动态调节（财务缺失时自动降权）
8. 信号评估回灌到阈值/置信度（半自动）
9. 云端部署方案（cron/systemd + 启动补偿）

## 验收标准
- 数据源回退后，财务因子可用率显著高于当前基线
- 报告内明确披露来源与可用性，不出现“隐式缺失”
- 置信度与数据质量、数据来源状态一致联动
